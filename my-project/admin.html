Operação ideal

Como isso funciona

1
Admin cria a guia

Você confirma o atendimento, define o valor e a guia privada já nasce pronta para o cliente seguir com o pagamento seguro.

2
Cliente recebe por WhatsApp

Sem cadastro obrigatorio. O envio inicial continua manual, com mensagem pronta e link seguro do proprio site.

3
Cliente escolhe a parcela

Antes do checkout, ele vê o valor atualizado do cartão e segue para a confirmação oficial no Mercado Pago.

4
Checkout oficial Mercado Pago

Os dados do cartão continuam fora do seu site. O checkout do Mercado Pago recebe a cobrança e processa o pagamento.

5
Email e painel em tempo real

Quando o pagamento aprovar, o webhook atualiza o painel, registra no histórico e pode disparar email para quem criou a cobrança.

Como o cliente paga

Ele escolhe a parcela na guia privada, confere a simulação e conclui no checkout oficial do Mercado Pago. O seu site não coleta dados sensíveis.

Como o dono recebe

O saldo cai na conta principal do Mercado Pago e o painel continua registrando os valores internos no backend.

Segurança primeiro

Os tokens sensíveis ficam no backend. No painel, o usuário logado salva apenas configurações operacionais, taxas e o email para receber o aviso.

Painel de faturamento

Visão rápida dos valores financeiros. Esses indicadores atualizam automaticamente conforme as aprovações do webhook.

O que importa aqui

Somente o token

Esta tela ficou enxuta. O único dado operacional daqui é o access token validado para a conta Mercado Pago ativa.

Segredo fora do navegador

Mesmo validando pela tela, o token não fica exposto no site. O frontend só mostra status e metadados mascarados.

Checkout continua protegido

A cobrança é gerada no backend e o cliente segue para o Checkout Pro do Mercado Pago. O site não recebe dados do cartão.

Troca simples de conta

Se precisar alternar entre contas, basta validar um novo token ou deletar o token salvo e cadastrar outro pela mesma tela.
